# Lessons Learned — Finestro

## Tech Stack Notes
- This is **Tailwind CSS v4** (not v3). There is NO `tailwind.config.ts/js`. All design tokens are set inside `index.css` using `@layer theme {}` via CSS custom properties.
- The compiled CSS file (`src/index.css`) is auto-generated by Tailwind v4 — **do not edit utility classes directly in it**. Add custom CSS via `@layer base`, `@layer utilities`, or additional CSS variables in `:root`.
- The project uses `@radix-ui/react-slot@1.1.2` style versioned imports in some components. Do not break these.
- Framer Motion must be installed via `npm install framer-motion` before use.

## Design System
- Brand accent color: Indigo `#6366F1`
- Brand dark: `#0F172A` (slate-900)
- Background: `#F8FAFC` (off-white, not pure white)
- Glassmorphism: `bg-white/70 backdrop-blur-xl border-slate-200/50`
- Shadows: multi-layered ambient shadows (see globals.css additions)

## Patterns
- All UI primitive components live in `src/components/ui/`
- Pages are swapped via a `currentPage` state in `App.tsx` — no router used
- Animations are added at the `App.tsx` level wrapping `renderPage()` output

## Gotchas
- PowerShell on Windows doesn't support `&&` as a command separator — run commands individually
- The dev server runs on port 3000 (configured in `vite.config.ts`)
- `index.css` is the compiled Tailwind output — add custom styles at the **bottom** of the file or in `src/styles/globals.css`
